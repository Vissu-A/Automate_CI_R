Pre-requisites to install Jenkins
---------------------------------
1. java
2. wget


Jenkins Installation steps for Ubuntu OS (apt)
----------------------------------------------

	1. Updating OS packges
	----------------------
		> sudo apt-get update						# to update the operating system libraries and packages.


	2. Installing Java
	-------------------
		> sudo apt-get install openjdk-8-jdk			# installing java-8.
		> java -version							# check the java version installed in system.


	3. Installing Linux wget package
	--------------------------------
		> sudo apt-get install wget


	4. Installing Jenkins
	----------------------

		https://pkg.origin.jenkins.io/				# Jenkins Software for all Operating Systems download index directory.
	
		Option-1: Long Term Support release (recommended)
		---------

			A LTS (Long-Term Support) release is chosen every 12 weeks from the stream of regular releases as the stable release for that time.
			It can be installed from the "debian-stable" repository.

			> sudo wget -q -O - https://pkg.jenkins.io/debian-stable/jenkins.io.key | sudo gpg --dearmor -o /usr/share/keyrings/jenkins.gpg

				Explanation of the above command:
				---------------------------------
				wget -q -O - https://pkg.jenkins.io/debian-stable/jenkins.io.key   # this is for adding the Jenkins repository key to our system.

				sudo gpg --dearmor -o /usr/share/keyrings/jenkins.gpg    # this is for converting the key into a format that apt can recognizes.


			> sudo sh -c 'echo deb [signed-by=/usr/share/keyrings/jenkins.gpg] http://pkg.jenkins.io/debian-stable binary/ > /etc/apt/sources.list.d/jenkins.list'

				Explanation of the above command:
				---------------------------------
				The [signed-by=/usr/share/keyrings/jenkins.gpg] portion in the above command ensures that apt will verify files in the repository using the GPG key that you just downloaded.

			> sudo apt-get update       						   # updating the packages to reflect the newly added Jenkins packages

			> sudo apt-get install jenkins						   # Installing Jenkins



		Option-2: Weekly release (Not recommended)
		---------
			
			A new release is produced weekly to deliver bug fixes and features to users and plugin developers. It can be installed from the debian
			repository.

			> sudo wget -q -O - https://pkg.jenkins.io/debian/jenkins.io.key | sudo gpg --dearmor -o /usr/share/keyrings/jenkins.gpg

			> sudo sh -c 'echo deb [signed-by=/usr/share/keyrings/jenkins.gpg] http://pkg.jenkins.io/debian binary/ > /etc/apt/sources.list.d/jenkins.list'

			> sudo apt-get update

			> sudo apt-get install jenkins



Jenkins Installation steps for Red Hat OS (yum)
-----------------------------------------------

	1. Updating OS packges
	----------------------
		> sudo yum update							# to update the operating system libraries and packages.


	2. Installing Java
	-------------------
		> sudo yum install java-1.8.0-openjdk-devel		# installing java-8.
		> java -version							# check the java version installed in system.


	3. Installing Linux wget package
	--------------------------------
		> sudo yum install wget


	4. Installing Jenkins
	----------------------

		https://pkg.origin.jenkins.io/				# Jenkins Software for all Operating Systems download index directory.
	
		Option-1: Long Term Support release (recommended)
		---------

			A LTS (Long-Term Support) release is chosen every 12 weeks from the stream of regular releases as the stable release for that time.
			It can be installed from the "debian-stable" repository.

			> sudo wget -O /etc/yum.repos.d/jenkins.repo https://pkg.jenkins.io/redhat-stable/jenkins.repo

			> sudo rpm --import https://pkg.jenkins.io/redhat-stable/jenkins.io.key

			> sudo yum update       						   	# updating the packages to reflect the newly added Jenkins packages

			> sudo yum install jenkins						   	# Installing Jenkins


		Option-2: Weekly release (Not recommended)
		---------
			
			A new release is produced weekly to deliver bug fixes and features to users and plugin developers. It can be installed from the debian
			repository.

			> sudo wget -O /etc/yum.repos.d/jenkins.repo https://pkg.jenkins.io/redhat/jenkins.repo

			> sudo rpm --import https://pkg.jenkins.io/redhat/jenkins.io.key

			> sudo yum update       						   	# updating the packages to reflect the newly added Jenkins packages

			> sudo yum install jenkins						   	# Installing Jenkins



Jenkins Installation steps for Amazon Linux OS (yum)
----------------------------------------------------

	1. Updating OS packges
	----------------------
		> sudo yum update							# to update the operating system libraries and packages.


	2. Installing Java
	-------------------
		> sudo yum install java-1.8.0-openjdk-devel				# installing java-11.
		> java -version									# check the java version installed in system.


	3. Installing Linux wget package
	--------------------------------
		> sudo yum install wget


	4. Installing Jenkins
	----------------------
	
		> sudo wget -O /etc/yum.repos.d/jenkins.repo https://pkg.jenkins.io/redhat-stable/jenkins.repo
		
		> sudo rpm --import https://pkg.jenkins.io/redhat-stable/jenkins.io.key

		> sudo yum upgrade

		> sudo yum install jenkins -y

		> sudo systemctl enable jenkins

		> sudo systemctl start jenkins

		> sudo systemctl status jenkins



Open jenkins in browser
-----------------------
go to your ec2 console and copy the instance public ip address.
now, open the browser and paste the copied public ip and port number of jenkins. (ex: http://public-ip:8080/) http://54.145.9.154:8080/

now, you can able to see the jenkins opened in browser, if you are not able to load jenkins and getting this site can't be reached any error.
then go to your ec2 instace security group and allow the jenkins port 8080 in the inbound rules.

for the first time it will ask to enter administrative password.
jenkins password will be stored in a file called "initialAdminPassword" at "/var/lib/jenkins/secrets/" directory.

read the the password using cat command "cat var/lib/jenkins/secrets/initialAdminPassword" copy the password and enter in the jenkins browser app.
Next, it will ask to create a admin account create it.


Start, stop, restart and check status of jenkins service
--------------------------------------------------------
> sudo systemctl start jenkins (or) sudo service jenkins start			# to start the jenkins

> sudo systemctl stop jenkins (or) sudo service jenkins stop			# to stop the jenkins

> sudo systemctl status jenkins (or) sudo service jenkins status			# to check the jenkins status

> sudo systemctl restart jenkins (or) sudo service jenkins restart		# to restart the jenkins
